## Component Responsibilities

### bin/hanif (Entry Point)
- **Input**: Command-line arguments
- **Output**: Exit code (0 = success, 1 = error)
- **Responsibilities**:
  - Environment setup
  - Top-level command routing
  - Version/help display
  - Error handling at highest level

### lib/commands/*.sh (Command Handlers)
- **Input**: Subcommands and arguments
- **Output**: Calls to functions, exit codes
- **Responsibilities**:
  - Subcommand parsing
  - Argument validation
  - Function orchestration
  - Command-specific help
  - Error handling for command category

### lib/functions/*.sh (Core Logic)
- **Input**: Validated parameters
- **Output**: Git operations, status codes
- **Responsibilities**:
  - Business logic implementation
  - Git operations
  - State management (stashing, etc.)
  - Return meaningful exit codes

### lib/utils/common.sh (Utilities)
- **Input**: Various (strings, commands, etc.)
- **Output**: Formatted output, validation results
- **Responsibilities**:
  - Logging and formatting
  - Input validation and sanitization
  - Helper functions
  - No side effects on git state

## Performance Considerations

### Optimization Points

1. **Single Fetch**: `gitupall` fetches once, not per branch
2. **Local Operations**: Prefer local git operations over remote
3. **Minimal Subshells**: Use built-in commands when possible
4. **Lazy Loading**: Commands load only when needed
5. **Caching**: Store repeated git queries in variables

### Scalability

- Handles repos with 100+ branches (gitupall)
- Works with multiple remotes
- Efficient for daily workflows
- No external dependencies (except git, bash)

## Security

### Safe Operations

- Never uses `eval` on user input
- Sanitizes branch names
- Validates git repository before operations
- Protects main/master/current branches
- Uses quotes around all variables
- Implements `set -euo pipefail`

### User Input Handling

```bash
# Bad (vulnerable)
eval "$user_input"

# Good (safe)
sanitized=$(sanitize_branch_name "$user_input")
git checkout -b "$sanitized"
```
